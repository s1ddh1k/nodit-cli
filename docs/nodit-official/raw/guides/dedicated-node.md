## Dedicated Node Service for Enterprise

Dedicated Node is a node infrastructure with allocated fixed resources available for exclusive use, providing more customized and consistent performance to your project. Unlike Elastic Node, Dedicated Node does not share resources with other clients, so you can fully control the capacity and performance of the node. It is a suitable product for projects providing high-traffic services that require reliable performance, enterprises that need control over node infrastructure, or projects requiring strict security and regulatory compliance.

The features provided through Endpoints are mostly the same as Elastic Node, and for some networks, additionally include calling permissions for sensitive APIs that cannot be used in Elastic Node for security reasons. As a management tool, it provides a separate management menu for Dedicated Node projects and dedicated Support Channels. Through this, you can check real-time metrics on node infrastructure, regular/irregular infrastructure work plans, and network issues in real-time, enabling stable service operation.

As the stability and flexible operational capability of nodes are recognized, Nodit's Dedicated Node is working with more and more projects. You can check various customer cases on the [Nodit Website](https://nodit.io) and [Blog](https://blog.nodit.io).

## Exploring Dedicated Node Features

### Dedicated Node Endpoint with No Usage or Throughput Limits

Since it uses fixed resources allocated to Dedicated Node projects, you can call Node Endpoints without usage or throughput limits within physically allowable ranges. Provides channels for monitoring node infrastructure performance.

### Dashboard

Provides a statistical Dashboard where you can easily check metrics related to usage patterns such as the number of requests through Dedicated Node, response time, and expected CU consumption. You can analyze traffic trends, throughput, and response speed without building separate admin functions to improve service quality and gain insights.







### Security Options Support for Building a Secure Operating Environment

You can control Node Endpoint access permissions and strengthen network security by setting Allowlists based on IP or Domain. When Allowlist is set, access requests from unlisted IPs or Domains to the Endpoint are blocked with a 403 Forbidden error response.



* Up to 25 Allowed IPs and Domains can be registered each.
* If both IP-based and Domain-based Allowlisting are enabled, only requests from Sources with matching both IP and Domain will be processed normally.

### Request Log for Debugging

Provides a Request Log menu where you can check all request history. You can check the complete call history including request and response details, or apply filters such as request time, Method, HTTP status code, JSON-RPC error code to monitor only request history that meets specific conditions separately.
